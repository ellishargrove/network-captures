STATIC ROUTES (FW-SRF-01)

192.168.10.0/24 → 192.168.10.2 (inbound from ISP segment, hairpin)
192.168.30.0/24 → null0 (guest VLAN blackhole if routing breaks)

These are fine. The guest blackhole has been there since 2017 - prevents guest traffic from
leaking to staff network if VLAN config breaks on the WLC.

---

BGP (not in use at SRF)

We don't run BGP. Single ISP, single uplink. If this changes (multi-homing discussion has come
up twice in the last year), I'll revisit. For now, static default to ISP gateway is sufficient.

---

NOTES ON ROUTE REDISTRIBUTION

The only redistribution happening is the OSPF → static default route. Nothing is being
redistributed into OSPF from connected or static - all internal routes are OSPF native.
There was a brief period in 2018 when a misconfigured redistribution was injecting connected
routes and bloating the LSDB. That's been cleaned up. If the LSDB starts growing unexpectedly
again, redistribution is the first thing to check.
